# 莲池竞渡 — 游戏拆解文档

## 一、游戏简介

**莲池竞渡**是一款基于桌游《骆驼大赛》(Camel Up, 2014 Spiel des Jahres) 改编的浏览器网页游戏。主题从埃及沙漠赛骆驼改为盛唐江南水乡的灵龟赛跑。玩家扮演观赛宾客，通过下注和布局锦囊在不确定性中做出最优决策，最终铜钱最多者封号"莲池伯"。

- **平台**：浏览器，纯前端，`file:///` 直接运行，零依赖
- **人数**：单人 vs 4名AI，或热座2-6人
- **时长**：约15-25分钟
- **技术栈**：原生 HTML + CSS + JavaScript（Canvas渲染），无框架、无构建工具

## 二、核心玩法拆解

### 2.1 基础循环

```
新一巡 → 玩家轮流行动 → 5支签抽完 → 验莲榜结算 → 下一巡 → ... → 龟至终点 → 揭榜
```

每一巡（回合）签筒内有7支签（6支普通色签+1支黑白逆行签）。玩家按铜钱数量排序轮流行动（铜钱少者先），直到所有签被抽出。

### 2.2 四种行动

| 行动 | 名称 | 收益模型 | 风险 |
|------|------|---------|------|
| 观莲下注 | 拿取回合押注券 | 押对第1名得5/3/2文，第2名1文 | 押错赔1文 |
| 布囊设局 | 放置锦囊(+1/-1) | 踩中即得1文，扭转赛道格局 | 每局仅1次，时机关键 |
| 摇签问路 | 从签筒抽签 | 签筹换1文（稳定收入） | 无直接风险，但消耗行动机会 |
| 一卦定局 | 全程竞猜鳌头/殿后 | 按预测顺序得8/5/3/2/1文 | 猜错赔1文 |

### 2.3 核心机制：叠罗汉

多只灵龟在同一格时形成叠罗汉。下方龟移动时驮着背上所有龟一起走。最上层龟视为最领先。这一机制创造了"搭顺风车"的戏剧性——被驮的龟无需自己的签被抽出就能前进。

### 2.4 逆转机制：锦囊+逆行龟

- **锦囊**：浮萍(+1) 让龟额外前进，漩涡(-1) 让龟后退到底部。领先龟踩中漩涡瞬间变垫底。
- **逆行龟**（玄冥/素影）：从终点出发反向移动，推动遇到的普通龟向前。黑白共用1颗骰子，每轮仅一只移动。

这三层系统（叠罗汉+锦囊+逆行龟）叠加产生高度不可预测的赛道变化，是游戏策略深度的核心来源。

### 2.5 策略深度分析

| 策略维度 | 说明 |
|---------|------|
| 信息不对称 | 先下注全程竞猜奖励高（8文）但信息少；后下注信息多但奖励递减 |
| 行动节奏 | 铜钱少的玩家先行，越穷越需要靠策略翻盘 |
| 空间博弈 | 锦囊放在领先龟前方可逆转格局，但可能误伤后来者 |
| 概率估算 | 签筒中剩余签的颜色分布影响各龟前进概率 |
| 对冲策略 | 同时押多个颜色分散风险 |

### 3.2 数据流（单向）

```
用户输入 → actionPanel.dispatch(type, payload)
         → store.history 记录
         → system handler 修改 store.state
         → subscriber 通知 UI 更新
```

### 3.3 状态管理

核心是 `src/store/store.js`（约100行），提供：

- **确定性PRNG**：`store.prng.next(min, max)` 替代 `Math.random()`，支持存档回放
- **事件派发**：`store.dispatch(type, payload)` → `store.register(type, handler, slice?)`
- **切片订阅**：`store.subscribeTo('player.hp', callback)` 精确到状态路径
- **历史记录**：`store.history` 记录所有离散事件，组合 seed 即可完整回放

### 3.4 状态切片

| 切片 | 文件 | 内容 |
|------|------|------|
| `state.race` | raceSystem.js | 8龟位置、叠罗汉、终点记录 |
| `state.round` | roundSystem.js | 巡编号、签筒状态、行动顺序 |
| `state.players` | playerSystem.js | 玩家数据、铜钱、锦囊状态 |
| `state.bets` | betSystem.js | 回合押注、全程竞猜记录 |
| `state.pouches` | pouchSystem.js | 锦囊位置和类型 |
| `state.log` | logSystem.js | 人类可读的行为日志 |
| `state.ai` | aiSystem.js | AI思考状态标记 |

### 3.6 关键架构约束

1. **file:/// 兼容**：无 ES modules、无打包工具、无外部依赖
2. **IIFE + window 全局导出**：每个 JS 文件自执行函数
3. **纯状态**：`store.state` 仅含可序列化数据（无DOM引用、无类实例）
4. **Config vs State 分离**：静态数值在 `config/`，动态值在 `store.state`
5. **确定性**：`Math.random()` 禁用，全部走 `store.prng.next()`

### 3.7 游戏循环

```
requestAnimationFrame(gameLoop)
  ├── store.tick()
  ├── aiSystem.update(state, dt)    # AI只有轮到时才决策
  ├── raceSystem.update(state, dt)  # 动画中间态
  ├── 检查 settle/game_over 标记
  └── render(state)
       ├── trackRenderer.render()
       ├── turtleRenderer.render()
       ├── roundInfo.render()
       ├── playerBar.render()
       ├── actionPanel.updateState()
       ├── historyLog.render()
       └── unactedPanel.render()
```

## 四、AI系统拆解

### 4.1 决策优先级

```
1. 持有锦囊 → 找关键格投放（搅局型权重高）
2. 领先龟明确 → 拿该色回合押注券
3. 赛程过半 → 下注全程竞猜（激进型更早）
4. 兜底 → 抽签（稳定收入）
5. 无选择 → 跳过回合
```

### 4.2 三种性格

| 性格 | 锦囊倾向 | 全程竞猜倾向 | 风格 |
|------|---------|------------|------|
| conservative（保守型） | 20% | 10% | 张员外、钱掌柜 |
| aggressive（激进型） | 10% | 40% | 李书生 |
| disruptor（搅局型） | 50% | 20% | 赵镖头 |

### 4.3 防冻结机制

- 阶段检查：定时器回调验证 `phase === 'player_turn'`
- try-catch 兜底：任何异常不阻止 `_thinking` 复位
- 结算/结束时强制 `_thinking = false`

## 五、视觉设计拆解

### 5.1 色彩系统

| 用途 | 颜色 | 色值 |
|------|------|------|
| 背景底 | 宣纸渐变 | `#f6f2ea → #e4dac8` |
| 主文字 | 墨色 | `#1a1010` |
| 强调色 | 朱砂红 | `#c04040` |
| 池水 | 淡绿 | `#d4e8d0 → #b8cfa8` |
| 铜钱 | 金色 | `#b48c32` |

### 5.2 字体系统

- 标题：`KaiTi, STKaiti`（楷体）
- 正文：`FangSong, STFangsong, SimSun`（仿宋）
- 按钮：`KaiTi, STKaiti, "Microsoft YaHei"`（楷体回退雅黑）

### 5.3 交互动效

- 按钮 hover：`translateY(-1px) scale(1.04)` + `brightness(1.08)`
- 按钮 active：`translateY(1px) scale(0.97)` + `brightness(0.94)`
- 过渡曲线：`cubic-bezier(.34, 1.56, .64, 1)`（微弹性）

莲池竞渡 — 游戏拆解文档

基于桌游《骆驼大赛》改编的网页游戏 | 2026年5月

一、游戏简介

莲池竞渡是一款基于桌游《骆驼大赛》(Camel Up, 2014 Spiel des Jahres) 改编的浏览器网页游戏。主题从埃及沙漠赛骆驼改为盛唐江南水乡的灵龟赛跑。玩家扮演观赛宾客，通过下注和布局锦囊在不确定性中做出最优决策，最终铜钱最多者封号"莲池伯"。

  • 平台：浏览器，纯前端，file:/// 直接运行，零依赖
  • 人数：单人 vs 4名AI，或热座2-6人
  • 时长：约15-25分钟
  • 技术栈：原生 HTML + CSS + JavaScript（Canvas渲染），无框架、无构建工具

二、核心玩法拆解

2.1 基础循环

新一巡 → 玩家轮流行动 → 7支签抽完 → 验莲榜结算 → 下一巡 → ... → 龟至终点 → 揭榜

每一巡（回合）签筒内有7支签（6支普通色签+1支黑白逆行签）。玩家按铜钱数量排序轮流行动（铜钱少者先），直到所有签被抽出。

2.2 四种行动

行动名称收益模型风险
观莲下注拿取回合押注券押对第1名得5/3/2文，第2名1文押错赔1文
布囊设局放置锦囊(+1/-1)踩中即得1文，扭转赛道格局每局仅1次，时机关键
摇签问路从签筒抽签签筹换1文（稳定收入）无直接风险
一卦定局全程竞猜鳌头/殿后按预测顺序得8/5/3/2/1文猜错赔1文

2.3 核心机制：叠罗汉

多只灵龟在同一格时形成叠罗汉。下方龟移动时驮着背上所有龟一起走。最上层龟视为最领先。这一机制创造了"搭顺风车"的戏剧性——被驮的龟无需自己的签被抽出就能前进。

2.4 逆转机制：锦囊+逆行龟

  • 锦囊：浮萍(+1) 让龟额外前进，漩涡(-1) 让龟后退到底部。领先龟踩中漩涡瞬间变垫底。
  • 逆行龟（玄冥/素影）：从终点出发反向移动，推动遇到的普通龟向前。黑白共用1颗骰子，每轮仅一只移动。

这三层系统（叠罗汉+锦囊+逆行龟）叠加产生高度不可预测的赛道变化，是游戏策略深度的核心来源。

2.5 策略深度分析

策略维度说明
信息不对称先下注全程竞猜奖励高（8文）但信息少；后下注信息多但奖励递减
行动节奏铜钱少的玩家先行，越穷越需要靠策略翻盘
空间博弈锦囊放在领先龟前方可逆转格局，但可能误伤后来者
概率估算签筒中剩余签的颜色分布影响各龟前进概率
对冲策略同时押多个颜色分散风险

3.3 状态管理

核心是 src/store/store.js（约100行），提供：

  • 确定性PRNG：store.prng.next(min, max) 替代 Math.random()，支持存档回放
  • 事件派发：store.dispatch(type, payload)
  • 切片订阅：store.subscribeTo('player.hp', callback) 精确到状态路径
  • 历史记录：store.history 记录所有离散事件，组合 seed 即可完整回放

3.6 关键架构约束

  1. file:/// 兼容：无 ES modules、无打包工具、无外部依赖
  2. IIFE + window 全局导出：每个 JS 文件自执行函数
  3. 纯状态：store.state 仅含可序列化数据（无DOM引用、无类实例）
  4. Config vs State 分离：静态数值在 config/，动态值在 store.state
  5. 确定性：Math.random() 禁用，全部走 store.prng.next()

四、AI系统拆解

4.1 决策优先级

1. 持有锦囊 → 找关键格投放（搅局型权重高）
2. 领先龟明确 → 拿该色回合押注券
3. 赛程过半 → 下注全程竞猜（激进型更早）
4. 兜底 → 抽签（稳定收入）
5. 无选择 → 跳过回合

4.2 三种性格

性格锦囊倾向全程竞猜倾向角色
保守型20%10%张员外、钱掌柜
激进型10%40%李书生
搅局型50%20%赵镖头

4.3 防冻结机制

  • 阶段检查：定时器回调验证 phase === 'player_turn'
  • try-catch 兜底：任何异常不阻止 _thinking 复位
  • 结算/结束时强制 _thinking = false

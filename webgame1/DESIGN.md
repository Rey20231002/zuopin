# 龟兔赛跑 · 莲池竞渡 — 网页版游戏设计文档

## 游戏概述

**类型:** 聚会桌游改编 / 竞速下注  
**人数:** 单人 vs AI（4名AI对手），或热座多人（2-6人）  
**时长:** 约15-25分钟  
**平台:** 浏览器网页游戏（Canvas渲染，file:///直接运行）  
**原型机制:** Camel Up (Steffen Bogen, 2014)  
**主题:** 中国古代 · 龟兔赛跑 · 莲池竞渡

时值盛唐，江南水乡。一年一度的**"莲池竞渡"**在城中最大的莲花池畔举行。五只来自各地的灵龟齐聚一堂，在荷叶连连的池塘赛道上一决高下。城中的文人墨客、商贾豪绅纷纷前来观赛，在龟背上押下自己的眼光与运气。

## 1. 游戏组件（数字化）

| 实体 | 原型 | 说明 |
|------|------|------|
| 灵龟 | 骆驼 | 5只（青/赤/黄/白/黑，取五行之色），每只有独立的颜色和位置 |
| 赛道 | 赛道 | 16段莲花池塘，第16段为终点（"摘莲台"） |
| 签筒 | 骰子×5 | 竹制签筒，内藏5支签，每支对应一色灵龟，签上刻点数1-3 |
| 回合押注券 | 分段赛下注券 | 每色3张共15张，绘制莲花图案，押注当前回合一甲的排名 |
| 全程竞猜签 | 全程竞猜卡 | 每玩家5支（对应5色），下注"鳌头"（全程冠军）或"殿后"（全程末位） |
| 锦囊 | 沙漠图板 | 每玩家1个，可放"浮萍(+1)"或"漩涡(-1)"于赛道上 |
| 铜钱 | 金币 | 每位玩家的资金，起始3枚 |

## 2. 核心机制

### 2.1 灵龟移动规则

1. 从签筒中抽出一支签，对应颜色的灵龟按签上点数前进
2. **龟背叠罗汉（核心机制）**：落点已有灵龟时，新到达的龟叠在背上。整叠移动时，背上所有龟一起被驮着走
3. **驮载规则**：一只龟背着其他龟时，带着背上所有龟一起移动（"顺风搭车"）
4. 叠在一起的龟中，**最上层视为最领先**（在赛道上位置更靠前）
5. 踩到锦囊格时触发效果（见2.3）

### 2.2 一回合流程

每个回合中，签筒中的5支签依次被抽出（每支各抽一次）。每位玩家在回合中轮流行动，直到5支签全部抽出，该回合结束。

轮到玩家时，从以下**4种行动**中选1种：

| # | 行动 | 名称 | 规则 |
|---|------|------|------|
| 1 | **领取押注券** | 观莲下注 | 从版图拿1张未持有的回合押注券。持有无上限，可重复拿同色。回合结束时按当前排名结算 |
| 2 | **投放锦囊** | 布囊设局 | 将唯一1个锦囊放在赛道空格上（选"浮萍+1"或"漩涡-1"）。不能放在第1格、有龟的格、已放锦囊的相邻格。有龟踩中该格时，锦囊主人立即获得1枚铜钱 |
| 3 | **抽取签筒** | 摇签问路 | 拿1张"签筹"，从签筒抽1支未抽的签，对应灵龟移动。回合结束时每张签筹换1枚铜钱 |
| 4 | **押全程竞猜** | 一卦定局 | 将1支竞猜签面朝下放入"鳌头"（总冠军）或"殿后"（末位）签筒。按放置顺序叠放，不可插队。可多次竞猜不同颜色 |

### 2.3 锦囊效果

- **浮萍 (+1)**："顺水推舟"——灵龟踩中后再前进1步。若踩中后有龟叠上，叠在上面的龟不触发效果
- **漩涡 (-1)**："触礁搁浅"——灵龟踩中后退1步。若后退格有龟，退到那一叠的**最下方**（即"背负"后面的龟）
- 锦囊触发后，主人得1枚铜钱，锦囊从赛道移除

### 2.4 回合结算

5支签全部抽出后（称为"一巡"）：

- **回合押注结算**（"验莲"）：
  - 所押颜色获第1名（"一甲"） → 得 5/3/2 枚铜钱（第1/2/3张押对的券，按押注先后）
  - 所押颜色获第2名（"二甲"） → 得 1 枚铜钱
  - 其余名次 → 赔 1 枚铜钱
- **签筹兑钱**：每张持有签筹换 1 枚铜钱
- 所有回合押注券和签筹放回版图，5支签放回签筒，开始新一巡

### 2.5 游戏结束

任意灵龟抵达"摘莲台"（越过第16格终点）时游戏立即结束：

1. 进行最后一次回合结算
2. 依次翻出"鳌头"签筒和"殿后"签筒中的竞猜签（按放置顺序）：
   - 猜错 → 赔 1 枚铜钱
   - 猜对 → 按预测顺序获奖：第1个猜对得 **8**枚，第2名 **5**枚，第3名 **3**枚，第4名 **2**枚，第5名及以后 **1**枚
3. **铜钱最多者获胜，封号"莲池伯"**

## 3. 视觉与美术风格

### 3.1 整体风格：中国古风 · 水墨淡彩

- **主色调**：宣纸底（米白偏黄 `#f5f0e8`）、墨色（`#3d3228`）、朱砂红（`#c04040`）
- **点缀色**：石青（`#5b7f95`）、藤黄（`#e8c44a`）、 lotus粉（`#e8c0b0`）
- **边框装饰**：回纹/云纹边框线条，毛笔手写体标题
- **字体**：标题使用楷体/行楷风格（`KaiTi, STKaiti, serif`），正文使用宋体/仿宋。系统自带中文字体即可

### 3.2 赛道（莲池）

Canvas绘制的横向池塘赛道：
- 底色为淡绿色池水，带有细微波纹纹理
- 16段赛道以莲花/莲叶分隔，每格有淡淡的石板纹路
- 第1格为"起莲台"（起点），第16格为"摘莲台"（终点），终点格有金色莲蓬标记
- 赛道两侧有垂柳/竹影装饰

### 3.3 灵龟

5只灵龟以**龟壳颜色+背部花纹**区分，造型圆润可爱：

| 颜色 | 名称 | 五行 | 龟壳花纹 |
|------|------|------|---------|
| 青色 | 青玄 | 木 | 竹叶纹 |
| 赤色 | 赤霄 | 火 | 火焰纹 |
| 黄色 | 黄枢 | 土 | 云雷纹 |
| 白色 | 白泽 | 金 | 水波纹 |
| 黑色 | 墨渊 | 水 | 星辰纹 |

叠在一起的龟在视觉上有明显的高度差（每层偏移几个像素），最上层的龟略微放大以示"领先"。

### 3.4 UI面板

- 面板底色为仿古纸卷/竹简质感（半透明宣纸色 `rgba(245,240,232,0.92)`）
- 按钮为木质纹理，仿古印章样式，hover时微微放大
- 铜钱显示用圆形铜钱图标（外圆内方）+ 数字
- 行动面板底部装饰有水墨山水暗纹

### 3.5 动画

- 抽签：签筒微微晃动，一支签飞出
- 龟移动：缓慢爬行动画（约0.8秒/格），背上叠着的龟一起晃动
- 锦囊触发：浮萍展开/漩涡旋转特效
- 回合结算：卷轴展开动画

## 4. 玩家体验设计

### 4.1 短/中/长期目标

| 时间尺度 | 目标示例 |
|----------|---------|
| **短期**（每次轮到玩家） | 选择最优行动，赚取/保住铜钱 |
| **中期**（每巡/回合） | 通过回合押注和锦囊布局获取回合收益 |
| **长期**（整场比赛） | 正确预测"鳌头"和"殿后"，赢得全局 |

### 4.2 进度反馈

- 每巡结束后显示"验莲榜"——当前排名变化和铜钱收支摘要
- 全程竞猜揭示时有"揭榜"悬念动画（逐支翻签）
- 灵龟接近终点时，"摘莲台"发出金色光晕
- 玩家铜钱数量实时显示，AI玩家铜钱变化可见

### 4.3 无口号式设计验证

- "下注有紧张感" → 玩家必须在看到赛道形势后快速决策；他人抽签会导致龟位变化，产生"被抢先"的紧迫感
- "叠龟有趣" → 被驮的龟无需自己抽签就能前进，产生"搭顺风车"的意外之喜；漩涡让领先龟突然变成最底层，产生戏剧性反转
- "策略深度" → 玩家需要在「安全拿1铜钱（抽签）」和「高风险高回报（全程竞猜）」之间权衡，同时管理锦囊投放时机

## 5. AI 对手设计（单人模式）

### 5.1 AI 身份

AI对手不再是抽象的"电脑"，而是有姓名的古代人物：

| 角色 | 身份 | 性格 | 下注风格 |
|------|------|------|---------|
| 张员外 | 城中富商 | 精明稳重 | 保守型——偏爱稳定收入，后期才押全程 |
| 李书生 | 赶考举子 | 书生意气 | 激进型——很早就押全程竞猜，敢赌 |
| 赵镖头 | 镖局武师 | 豪爽莽撞 | 搅局型——善用锦囊制造意外 |
| 钱掌柜 | 当铺老板 | 精打细算 | 保守型——数学期望优先，不感情用事 |

### 5.2 AI 难度

| 难度 | 名称 | 行为特征 |
|------|------|---------|
| 简单 | "路过看客" | 随机选择行动，偶尔做出合理选择 |
| 普通 | "常客熟手" | 基于启发式规则决策（推荐默认） |
| 困难 | "莲池老手" | 评估概率和期望收益后决策 |

### 5.3 AI 决策逻辑（普通难度）

按以下优先级评估：

1. 持有锦囊时，优先找关键位置投放（领先龟前方、叠龟密集区域）
2. 当前回合领先龟明确时，拿取该颜色的回合押注券
3. 若签筒剩余未抽签 > 1 且铜钱 > 0，抽签（稳定收入）
4. 比赛过半（>50%回合已过），开始下注全程竞猜
5. 无更好选择时抽签

## 6. 模式设计

### 6.1 开始页面

古风标题画面：毛笔大字"莲池竞渡"，背景为水墨莲花池。

两个大按钮（印章样式）：
- **「独坐观莲」** — 单人 vs 4名AI对手（张员外、李书生、赵镖头、钱掌柜）
- **「群贤共弈」** — 热座多人（2-6人），同一设备轮流操作

单人模式可选AI难度。热座模式中，每位玩家操作前显示过渡遮罩："请将席位交予 玩家X"。

### 6.2 热座多人

- 每位玩家的竞猜签对他人隐藏（显示"?"封印标记）
- 玩家操作时，他人的竞猜信息被遮挡
- 回合结算时展示信息公开后的结果
- 全程竞猜揭示时，按玩家顺序逐一"揭榜"

## 8. 游戏流程状态机

```
START → 开始页面（标题+模式选择）
  → INIT（抽签定初始龟位、各玩家发3枚铜钱）
  → ROUND_START（新一巡开始，5签入筒）
    → PLAYER_TURN（当前玩家行动）
      → 选行动 → 动画演出 → 下一位玩家
    → ROUND_SETTLE（5签全抽出）
      → 验莲（结算回合押注）→ 兑签筹（签筹换钱）
  → CHECK_END（有龟至摘莲台？）
    → 否 → ROUND_START
    → 是 → FINAL_REVEAL（揭榜：翻鳌头/殿后签）→ 封号"莲池伯" → GAME_OVER
```

## 9. 遵循设计原则自查

| 原则 | 达成方式 |
|------|---------|
| 1. 直觉交互 | 池塘赛道=水+石阶、签筒=抽签动画、叠龟=视觉堆叠高度差、铜钱=圆形图标 |
| 2. 组合创意 | 锦囊(+1/-1) × 灵龟叠罗汉 = 不可预测的局势反转，无新增实体 |
| 3. 核心循环 | 观察赛道→下注/抽签→揭晓结果→调整策略，每巡可重复 |
| 4. 目标层级 | 短期（选行动）、中期（押一甲/二甲）、长期（押鳌头/殿后）清晰 |
| 5. 反退化策略 | 先押全程奖励高但信息少；回合押注安全但收益低；抽签稳定但缓慢。无单一统治策略 |
| 6. 玩家代理 | 每巡每个决策直接影响铜钱收入，锦囊投放改变赛道格局 |
| 7. 非口号 | 所有"有趣"描述均有具体机制支撑（见4.3节） |
| 8. PEGI 7 | 卡通灵龟、铜钱代币、封号奖励、无暴力元素 |
